关于预测信号异常为0和1的问题

Q: When I use MIL to tackle video anomaly detection task, I got a weird outcome that all scores of all positive bags and negative bags are closed to 0. But I expect that positive instances are likely to be 1. I use hinge loss as my loss function, and aggregation function is maximum. Can you help analyse it?

A: It sounds like you're running into a problem with your loss function and aggregation function. In the case of MIL, the idea is to take a collection of instances (called a bag) and determine whether the bag contains any positive instances. To do this, you typically want to use a loss function that encourages the model to output higher values for bags that contain positive instances and lower values for bags that do not contain positive instances.

The maximum aggregation function, on the other hand, takes the maximum value of all the instances in the bag and uses that as the bag's final output. This can lead to problems when the bag contains both positive and negative instances, because the maximum function will always output the maximum value, regardless of whether it comes from a positive or negative instance.

To solve this problem, you may want to try using a different loss function and aggregation function. For example, you could try using the sum or mean aggregation function, which will take into account all the instances in the bag, rather than just the maximum value. You could also try using a loss function that is better suited to MIL tasks, such as the binary cross-entropy loss.

Overall, the specific loss function and aggregation function you use will depend on the details of your task and the requirements of your model. It's worth experimenting with different combinations to find the one that works best for your problem.

损失函数的选择和不下降的问题

Q: I tried BCE loss. However the loss would not decrease. I have no idea what happened.

A: If the loss is not decreasing when using the binary cross-entropy (BCE) loss function, it could be due to a variety of reasons. Some possible causes include:

  • The model may not be learning at all, which could be due to a poor choice of model architecture, a lack of training data, or incorrect model hyperparameters.
  • The model may be overfitting to the training data, which can cause the loss to stop decreasing even if the model is still learning. This can happen if the model is too complex for the amount of training data available, or if the training data is not representative of the true distribution of the data.
  • The learning rate may be too high, which can cause the model to oscillate or diverge, instead of converging on a minimum loss value.

To diagnose the problem, it may be helpful to visualize the loss during training using a tool like TensorBoard. This can give you a better idea of what's happening during training, and can help you identify potential issues with the model or the training process.

If the loss is not decreasing, you may want to try adjusting the model architecture, the training data, the learning rate, or other hyperparameters to see if that helps the model learn more effectively. It's also important to make sure that the training data is representative of the true distribution of the data, as this can affect the model's ability to generalize to new, unseen examples.

如何向多示例学习加入先验知识

Q: I want to add some prior information to the MIL training. The input samples are extracted features, and I can compute saliency of each instance in a bag. How can I accomplish it?

A: To incorporate prior information into a multiple instance learning (MIL) model, you can try using a regularization term in the loss function that encourages the model to use the prior information when making predictions.

For example, suppose you have a bag of instances, and you have computed the saliency of each instance in the bag. You can then use this saliency information as a regularization term in the loss function, so that the model is encouraged to give more weight to instances with higher saliency when making predictions for the bag.

To do this, you would need to define a regularization term that takes into account the saliency of each instance in the bag. For example, you could define the regularization term as the sum of the saliency values of all the instances in the bag, multiplied by a regularization strength hyperparameter. This would encourage the model to give more weight to instances with higher saliency, and would help the model use the prior information more effectively when making predictions.

In general, the specific form of the regularization term will depend on the details of your task and the requirements of your model. It's worth experimenting with different regularization terms to see what works best for your problem.

关于可视化工具

Q: when I tried to visualize my training data information, I found it extremely complicated to redefinate functions via matplotlib. Any better methods to visualize my tensor data which are more handy (for me, even tensorboard is kind of complicated)?

A: If you find it difficult to visualize your data using tools like matplotlib or TensorBoard, there are other options you can try. For example, you can use a tool like TensorFlow Playground (https://playground.tensorflow.org/) to interactively visualize and explore your data. This can be a useful way to get a better understanding of your data, and can help you identify patterns and trends that may not be immediately obvious.

Another option is to use a tool like Embedding Projector (https://projector.tensorflow.org/) to visualize high-dimensional data in a lower-dimensional space. This can be particularly useful for visualizing data that has many features or dimensions, as it can help you understand the relationships between different features and identify clusters or patterns in the data.

Overall, there are many different tools and techniques you can use to visualize your data, and the best one for you will depend on the specific details of your data and the goals of your visualization. It's worth experimenting with different tools and techniques to see what works best for your problem.
